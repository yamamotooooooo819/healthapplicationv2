# UI 設計書 — 健康管理アプリ(MVP / 第1段階)

このファイルは画面・UI の設計方針。`CLAUDE.md` の設計判断、`api-design.md` のAPI仕様に従う。
モックの壁打ちで合意した方針を記録する。

---

## 0. 全体方針

- **スマホ主役のレスポンシブ設計**。食事は「食べた直後にその場でスマホ入力」が主シーン。
  まずスマホ縦長レイアウトを固め、PC はその拡張(中央寄せ・余白拡大)として扱う。
- 画面は3つが核: **ログイン / メイン画面 / 食品検索・選択**。
- メイン画面が中心。日付を切り替えて過去も閲覧する。

### 画面遷移
```
ログイン ──(認証成功)──▶ メイン画面 ──(＋追加)──▶ 食品検索・選択
                                  ◀──(選んで戻る)──
```

---

## 1. メイン画面(最重要)

スマホ縦長。上から順に以下を縦に積む。

### (a) 日付切り替えバー(最上部)
- 中央に日付(例「5月24日 (土)」)、その下に補助ラベル(「今日」など)。
- 左右に前日/翌日へ移動する矢印ボタン。
- 過去日付に移動すると、その日の記録を表示する。
- 対応API: 表示中の日付を `GET /api/summary/daily?date=` と
  `GET /api/meal-entries?date=` に渡す。

### (b) PFC 集計エリア
- **数値 + ドーナツグラフの両方**で見せる(決定事項)。
- ドーナツグラフで P/F/C の比率を視覚化。中央に1日の総カロリー(kcal)。
- グラフ右に P / F / C それぞれの数値(g)を色付きの凡例とともに表示。
- 配色: **P = 青(#378ADD) / F = 橙(#EF9F27) / C = 緑(#1D9E75)**(決定事項)。
- 対応API: `GET /api/summary/daily` のレスポンス
  (`total` を中央と凡例に、比率はグラフに)。

### (c) 食事リスト(朝・昼・晩・間食)
- 4区分を縦に並べる。各区分は「見出し行 + 明細カード」。
- 見出し行: 区分アイコン + 区分名 + その区分の小計 kcal。
  - **見出しの右上に「＋」追加ボタン**(決定事項)。押すと食品検索・選択へ。
  - 「どの区分に追加するか」が明確になる配置。
- 明細カード: その区分の食品を1行ずつ(食品名 + 量g + kcal)。
  - 同一食品が重複して並ぶことがある(おかわり等。設計判断7)。
- **記録ゼロの区分は「まだ記録がありません」の空状態を表示**(4区分は常に表示)。
  集計APIが0を返す前提(api-design.md §5)と一致。

### (d) 食品行のタップ操作
- 食品行をタップすると **「量の編集」と「削除」のみ** を表示する(決定事項)。
  - 量の編集: `quantity_g` を変更し、サーバー側で PFC スナップショットを再計算。
  - 削除: 論理削除(設計判断9)。
- **名前の変更は提供しない**(決定事項)。
  理由: 食品名は全ユーザー共有の `foods` マスタの値であり、
  1ユーザーが書き換えると他人の記録の表示まで変わるため。
  自分用に調整した食品が欲しい場合は `POST /api/foods`(ユーザー食品登録)で対応。
- 量編集・削除の実装は**第2段階**。MVPのモックでは入口だけ描いてよい。

---

## 2. 食品検索・選択画面

メイン画面の「＋」から開く(全画面 or モーダル)。

- 上部に検索ボックス。入力に応じて `GET /api/foods/search?q=` を呼ぶ。
- 結果は20件ずつ・一致度順(api-design.md §3)。スクロールで追加読み込み。
- 各結果に食品名 + 100gあたりの簡易栄養情報。
- 食品を選ぶ → 量(g)を入力 → 「追加」でメインに戻る。
  - 一括登録方式(設計方針4)に合わせ、複数選んでまとめて追加できるとなお良い。
  - 対応API: `POST /api/meal-entries`(新規)または
    `POST /api/meal-entries/{id}/items`(既存区分への追加)。
- 「自分の食品を登録」への導線(`POST /api/foods`)もここに置けるとよい。

## 4. 実装上の注意(Claude Code への指示)

- まずスマホ幅(～380px)で破綻しないレイアウトを作る。PC は中央寄せで拡張。
- PFC・kcal などの表示数値は丸める(小数の桁が暴れないように)。
- 空状態(記録なし・検索結果なし)を必ず用意する。
- 配色 P=青 / F=橙 / C=緑 はアプリ全体で統一する。
- 量編集・削除・ユーザー食品登録の UI 実装は第2段階。MVP は入口の設置まで。
